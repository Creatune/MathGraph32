Spécificités de la version java

Présentation du logiciel ››

La version java possède toutes les possibilités offertes par la version Windows à l'exception de la possibilité de coller une figure dans un document en gardant un lien OLE sur ce document.

Il est toujours possible de coller une figure MathGraph32 dans un document après l'avoir copiée dans le presse-papier mais un double clic sur la figure ne lancera pas la version java de MathGraph32.

Il est donc conseillé de conserver une copie de la figure que vous collez dans le document dans le répertoire contenant le document au cas où vous souhaiteriez la modifier plus tard.


Mais la version java possède aussi de nouveaux atouts :


  • MathGraph32 devient ainsi multi plateforme.
  • La version java a été repensée et est plus ergonomique.
  • Une nouvelle icône apparaît : qui permet de modifier tout objet graphique en faisant apparaître le menu correspondant.
    Inutile donc dans la version java de dire quel est le type d'objet que l'on souhaite modifier. Un clic sur l'objet avec cet outil suffit.
  • MathGraph32 java permet de mettre un calcul quelconque dans tout champ où une valeur numérique est attendue. Par exemple on peut créer un cercle de rayon 2*OA où OA est une longueur qui a été précedemment mesurée. Avec la version Windows il aurait fallu d'abord créer un calcul nommé par exemple a et contenant comme formule  2*OA puis créer une cercle de rayon a.
  • MathGraph32 java permet de charger toutes les figures créées avec la version Windows.
  • Les caractères grecs et spéciaux sont maintenant gérés de façon native : quand vous entrez du texte dans un éditeur où certains caractères spéciaux sont permis une boîte de dialogue s'ouvre automatiquement pour vous permettre d'entrer les caractères spéciaux.
  • MathGraph32 java est personnalisable : On peut n'autoriser que certains outils. Le logiciel peut ainsi s'adapter aux désirs des enseignants.
