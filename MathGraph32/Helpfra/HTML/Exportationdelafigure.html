Exportation de la figure

Le menu Fichier - Exporter vous permet d'enregistrer la figure en cours sous différents formats.


  • Un fichier d'image PNG.
  • Un fichier d'image JPEG
  • Une page web dynamique utilisant une applet (html) qui vous permettra de mettre votre figure en ligne.
  • Une page Web dynamique utilisant uniquement JavaScript (html 5) qui vous permettra de mettre votre figure en ligne sans utiliser d'applet. Cette option vous permettra d'obtenir une page html visible dans la plupart des navigateurs récents, y compris les périphériques mobiles (tablettes, smartphones). Une autre avantage est que les problèmes de sécurité relatifs aux applets seront évités.
  • Le code LaTeX Tikz permettant d'inclure la figure dans un document LaTeX.


Le menu Fichier - Exporter avec unité dans un fichier PNG vous permet d'exporter votre figure dans un fichier PNG spécial en spécifiant une longueur en centimètres pour la longueur unité de la figure. Une fois cette image incorporée dans un traitement de texte, elle aura exactement les dimensions souhaitées (à condition qu'elle tienne dans la page dans laquelle elle est collée).


Compléments sur l'exportation dans une page web dynamique utilisant uniquement JavaScript :


La sortie graphique se fait dans un SVG (graphique vectoriel).

L'emploi de l'explorer de Microsoft est déconseillé pour visualiser les pages générées par l'explorer gère mal les SVG à l'heure actuelle (problèmes de clipping).

Lorsque vous éditez la page html générée par MathGraph32, vous verrez un code analogue à celui-ci dans la fonction go() :

var st = "Code de la figure en base 64";

mtg32App.addDoc("svg", st);

Le premier paramètre "svg" de la deuxième ligne fait référence au SVG défini en pas de la page ayant pour Id "svg".

Si vous voulez insérer deux figures dans une même page, vous devrez créer dans la page un autre svg ayant pour Id par exemple "svg2" (qui doit être contenu dans une section div) puis ajouter deux autres lignes dans go() à la suite des deux précédentes :

st = "Code de la deuxième figure en base 64";

mtg32App.addDoc("svg2", st);


Bien sûr vous pouvez insérer plus de 2 figures dans une même page html.


Pour obtenir le code base 64 d'une figure il suffit, lorsque cette figure est ouverte dans MathGraph32, d'utiliser le menu Edition Copier le code de la figure (pour html). Le code de la figure est alors copié dans le presse-papier et vous pouvez ensuite le coller dans votre page html entre les guillemets suivant st =.


Il est aussi possible de récupérer le code Base 64 d'une figure exportée dans une page html pour l'éditer dans MathGraph32 en utilisant le menu Fichier - Nouvelle figure par code.


Compléments sur l'exporation du code LaTeX Tikz :


Il y deux façons d'exportation d'exporter le code LaTeX de la figure :

  • Par le menu Fichier - Exporter Code LaTeX Tikz :

Une boîte de dialogue s'ouvre :

Si la figure comporte une longueur unité et si la case à cocher Utiliser longueur unité est cochée, celle-ci sera représentée par défaut par 1 cm.

Mais vous pouvez spécifier un coefficient d'agrandissement-réduction. A chaque fois que ce coefficient est modifié, le code doit être régénéré.

Si la figure ne comporte pas de longueur unité ou si la case à cocher Utiliser longueur unité n'est pas cochée , MathGraph32 s'arrange pour que 1 cm sur votre écran soit représenté par 1 cm, avec là-aussi la possibilité d'appliquer une coefficient d'agrandissement-réduction.

Vous générez le code en cliquant sur le bouton Générer code Tikz. Il apparaît alors dans le champ d'édition.

En cliquant sur le bouton Copier code complet, vous copiez le code contenu de l'éditeur dans le presse-papier. Si vous avez sélectionné une partie du code, c'est la partie sélectionnée qui est copiée dans le presse-papier.

En cliquant sur le bouton Copier code figure vous copiez uniquement la description de la figure sans les en-tête de document.

Si la figure comporte une image de fond ou des images, celles-ci sont enregistrées dans des fichiers png dont le nom commence par le nom de la figure et est suivi d'un numéro (zéro pour l'image de fond s'il y en a une). Ces fichiers doivent être présents dans le dossier contenant le fichier LaTeX lors de la compilation pour qu'elles apparaissent dans le document LaTeX après la compilation.

  • Par menu Edition - copier code LaTeX Tikz.

Le code est alors directement copié dans le presse-papier. Le coefficient d'agrandissement-réduction appliqué est celui choisi dans le menu Préférences - Presse-papier.


Compléments sur l'exportation dans une page web dynamique utilisant une applet :


Lorsque le menu Fichier - Exporter dans - Page web dynamique est activé, une boîte de dialogue s'ouvre.

Dans la partie du haut, vous pouvez entrer un titre pour la figure, le nom de l'auteur et les dimensions en pixels qu'occupera la figure (applet).

Dans la partie du bas, l'onglet Texte Html vosvg.us permet d'entrer du texte devant figurer avant et après la figure.

L'onglet Options vous permet de choisir les parties de l'applet qui devront être présentes.

Si vous laissez décochée la case Inclure le Jar dans l''export, l'appel de l'applet contenu dans le code html se fera en ligne.

Si, par contre, vous cochez la case  Inclure le Jar dans l''export, le fichier jar de l'applet sera enregistré dans le même dossier que la page html. Il faudra choisir cette option si vous voulez utiliser cette figure sans être connecté à Internet.

Vous devrez ensuite choisir l'emplacement où le fichier htm (et éventuellement le fichier jar) doit être enregistré.

Une fois l'exportation finie, si votre machine le permet, le navigateur internet par défaut sera lancé avec le fichier html généré.


Deux types d'exportations sont proposées :


  • Mode visualisation (mode par défaut) : La figure apparaîtra dans la page internet et il sera possible de capturer un point mobile ou de déclencher une macro. Mais il ne sera pas possible de modifier la figure en ligne ni de l'enregistrer. Cette applet est plus légère donc plus rapidement chargée et ne provoque pas l'affichage d'un message de sécurité.


  • Mode création : La figure apparaître dans une fenêtre équivalente à l'application MathGraph32 en ligne et il vous sera possible de modifier et enregistrer la figure en ligne. L'applet utilisée est plus lourde et donc plus longue à charger et elle provoque un avertissement de sécurité (à accepter).
